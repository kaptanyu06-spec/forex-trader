# ระบบวิเคราะห์ Forex: ข่าว + เทคนิค + ประเมินความเสี่ยง

ระบบนี้ดึงข่าวที่เกี่ยวข้องกับสกุลเงินที่คุณติดตาม วิเคราะห์ความรู้สึกของข่าว (sentiment)
วิเคราะห์กราฟด้วย technical indicators (MA, RSI, MACD, ATR, Bollinger Bands)
และช่วยคำนวณขนาดออเดอร์ตามหลัก risk management เพื่อ**ประกอบการตัดสินใจ**
**ระบบนี้ไม่ใช่คำแนะนำการลงทุน และไม่ส่งคำสั่งเทรดจริงให้อัตโนมัติ** — คุณเป็นผู้ตัดสินใจเข้าเทรดเอง

## 2. วิธีติดตั้ง (ทำครั้งเดียว)

### ขั้นตอนที่ 1: ติดตั้ง Python
ถ้ายังไม่มี Python ในเครื่อง ดาวน์โหลดจาก https://www.python.org/downloads/
(เลือกเวอร์ชัน 3.10 ขึ้นไป)

### ขั้นตอนที่ 2: ติดตั้ง Library ที่จำเป็น
เปิด Terminal/Command Prompt ไปที่โฟลเดอร์ `forex_news_bot` แล้วรัน:

```bash
pip install -r requirements.txt
```

### ขั้นตอนที่ 3: สมัคร NewsAPI (ฟรี)
1. ไปที่ https://newsapi.org/register
2. สมัครด้วยอีเมล จะได้ API Key ทันที (ฟรี 100 requests/วัน)
3. เปิดไฟล์ `config.py` แล้วแก้บรรทัด:
   ```python
   NEWSAPI_KEY = os.environ.get("NEWSAPI_KEY", "ใส่_API_KEY_ของคุณที่นี่")
   ```
   เปลี่ยน `"ใส่_API_KEY_ของคุณที่นี่"` เป็น API Key ที่ได้มา

### ขั้นตอนที่ 4: ปรับแต่งคู่เงินที่ต้องการติดตาม (ไม่บังคับ)
เปิดไฟล์ `config.py` แก้ตัวแปร `WATCHED_PAIRS` ให้ตรงกับคู่เงินที่คุณเทรด

## 3. วิธีรันระบบ

มี 2 แบบ ให้ผลวิเคราะห์เหมือนกัน:

**แบบที่ 1: หน้าเว็บ Dashboard (แนะนำ — ดูง่ายกว่า)**
```bash
streamlit run dashboard.py
```
browser จะเปิดหน้า Dashboard อัตโนมัติ (http://localhost:8501)
กดปุ่ม "รันวิเคราะห์ใหม่" ที่แถบซ้ายเพื่อดึงข้อมูลล่าสุด มีกราฟราคา+MA
และย้อนดูผลวิเคราะห์เก่าทุกครั้งได้

**แบบที่ 2: รันใน Terminal**
```bash
python main.py
```

**แบบที่ 3: รันอัตโนมัติ + Paper Trading (ขั้นพิสูจน์กลยุทธ์)**
```bash
python scheduler.py
```
เปิดหน้าต่างนี้ทิ้งไว้ ระบบจะวิเคราะห์เองทุก 2 ชั่วโมง:
- เมื่อเกิดสัญญาณ BUY/SELL -> บันทึกเป็น **เทรดจำลอง** (paper trade) พร้อม SL/TP
- รอบถัดไปเช็คราคาจริงว่าเทรดจำลองชนะ (แตะ TP) หรือแพ้ (แตะ SL)
- สถิติสะสมดูได้ในหน้า Dashboard (อัตราชนะ, ผลตอบแทนจำลอง)
- ตั้งค่า Telegram ใน `config.py` แล้วจะแจ้งเตือนเข้ามือถือทุกครั้งที่เปิด/ปิดไม้
  (วิธีสมัครบอทอ่านที่หัวไฟล์ `notifier.py` — ใช้เวลา ~2 นาที ฟรี)

**แบบอัตโนมัติเต็มรูปแบบ (ที่ใช้อยู่ปัจจุบัน — รันบนคลาวด์ ฟรี)**:
ระบบรันบน **GitHub Actions** ทุก 2 ชั่วโมง ที่ repo ส่วนตัว
https://github.com/kaptanyu06-spec/forex-trader — **ไม่ต้องเปิดคอมเลย**
ผลทุกอย่างแจ้งเข้า Telegram (@KT_Trader_bot) และสมุดเทรดจำลอง
(`output/paper_trades.json`) ถูกบันทึกกลับเข้า repo ทุกรอบ

- ดูประวัติการรัน: เข้า repo บนเว็บ -> แท็บ **Actions**
- สั่งรันทันที: แท็บ Actions -> "Forex Analyzer" -> ปุ่ม **Run workflow**
- รหัสลับ (NewsAPI, Telegram) เก็บใน GitHub Secrets (เข้ารหัส) + `secrets_local.py`
  ในเครื่อง ซึ่ง**ไม่ถูกอัปโหลด** (.gitignore กันไว้)
- ดึงข้อมูล paper trade ล่าสุดมาดูใน Dashboard ในเครื่อง: `git pull` ก่อน

ตัวสำรองในเครื่อง: ไฟล์ launchd `~/Library/LaunchAgents/com.forextrader.scheduler.plist`
ยังอยู่แต่**ปิดใช้แล้ว** (กันรันซ้ำซ้อนกับคลาวด์) — ถ้าอยากกลับมาใช้:
`launchctl load ~/Library/LaunchAgents/com.forextrader.scheduler.plist`

ข้อจำกัด: เวลารันบนคลาวด์อาจเลื่อนจากตารางไม่กี่นาที และตลาดปิดเสาร์-อาทิตย์

**เป้าหมายขั้นนี้**: เก็บ paper trade อย่างน้อย 1-3 เดือน แล้วเทียบอัตราชนะ/ผลตอบแทน
กับผล backtest — ถ้าใกล้เคียงกันจึงค่อยพิจารณาขั้น demo account ผ่าน MT5 ต่อไป

ระบบจะ:
1. ดึงข่าวล่าสุดของแต่ละสกุลเงิน (ข้ามอัตโนมัติถ้ายังไม่ได้ใส่ NewsAPI key)
2. คำนวณคะแนน sentiment (-1 ถึง +1) ของแต่ละสกุลเงิน
3. เปรียบเทียบสองสกุลเงินในคู่เดียวกัน บอกว่าข่าวเอียงไปทางไหน
4. ดึงราคาย้อนหลัง แล้ววิเคราะห์กราฟด้วย technical indicators
5. ประเมินว่าสัญญาณนี้ "น่าเชื่อถือแค่ไหน" (ต่ำ/ปานกลาง/สูง)
6. พิมพ์สรุปผลหน้าจอ + บันทึกไฟล์ JSON ละเอียดไว้ในโฟลเดอร์ `output/`
7. โชว์ตัวอย่างการคำนวณขนาด position ตามการบริหารความเสี่ยง

นอกจากนี้ยังทดสอบแต่ละส่วนแยกได้:
```bash
python price_fetcher.py        # ทดสอบดึงราคา (ต้องต่อเน็ต)
python technical_analyzer.py   # ทดสอบ indicator ด้วยข้อมูลจำลอง (ไม่ต้องต่อเน็ต)
```

## 4. วิธีอ่านผลลัพธ์

### ส่วนข่าว (sentiment)
- **Net sentiment score**: ค่าบวก = ข่าวหนุนสกุลเงินตัวแรก (base) แข็งค่าขึ้น, ค่าลบ = หนุนสกุลเงินตัวหลัง (quote)
- **Bias**: สรุปเป็นภาษาที่อ่านง่ายว่าโน้มไปทางไหน
- **ความน่าเชื่อถือสัญญาณ**: บอกว่าควรเชื่อสัญญาณนี้แค่ไหน (ยิ่งข่าวน้อย/คะแนนอ่อน ยิ่งไม่ควรเชื่อเยอะ)

### ส่วนเทคนิค
- **MA Crossover**: เส้นค่าเฉลี่ยเร็ว (20 แท่ง) เทียบเส้นช้า (50 แท่ง) — เร็วอยู่เหนือช้า = แนวโน้มขาขึ้น
- **RSI**: เกิน 70 = ซื้อมากเกิน (ระวังย่อ), ต่ำกว่า 30 = ขายมากเกิน (อาจเด้ง)
- **MACD**: วัดโมเมนตัม — MACD เหนือเส้น signal = โมเมนตัมขาขึ้น
- **Bollinger Bands**: ราคาทะลุกรอบบน/ล่าง = ราคา "ตึง" เกินไปด้านนั้น
- **ระยะ SL แนะนำจาก ATR**: ระยะ stop-loss ที่เหมาะกับความผันผวนปัจจุบัน (1.5 เท่าของ ATR)
- **คะแนนโหวต (technical_vote)**: รวมเสียงจาก 4 indicators ได้ค่า -4 ถึง +4
  — ตั้งแต่ +2 = BULLISH, ตั้งแต่ -2 = BEARISH, ระหว่างนั้น = NEUTRAL

ค่าทุกตัว (จำนวนแท่ง MA, ระดับ RSI, timeframe ฯลฯ) ปรับได้ในไฟล์ `config.py` หมวด 4

### ข้อสรุปรวม (สัญญาณสุดท้าย)
ระบบใช้กลยุทธ์ **"ตามเทรนด์ + รอย่อ"** (ผ่านการ backtest เทียบแล้วดีกว่าระบบโหวตเดิมทุกคู่เงินบน 1h):
- **ทิศทาง**: MA และ MACD ต้องเห็นตรงกัน (ขัดกัน = ไม่เทรด)
- **จังหวะเข้า**: รอราคาย่อสวนเทรนด์ก่อน (ขาขึ้นรอ RSI < 50, ขาลงรอ RSI > 50)
- **ข่าวเป็นตัวกรอง**: ชี้ทางเดียวกัน = ความเชื่อมั่น "สูง", เป็นกลาง/ไม่มีข่าว = "ปานกลาง",
  ชี้สวนทาง = **WAIT** เสมอ (ปลอดภัยไว้ก่อน)
- ทุกสัญญาณ BUY/SELL จะแนบ **ระยะ SL แนะนำ (pip)** และ**ขนาด lot** ที่คำนวณจาก risk 1% ให้เสมอ

การเห็น **WAIT ทุกคู่เป็นเรื่องปกติ** — ตลาดส่วนใหญ่ไม่มีจังหวะที่ดีตลอดเวลา
ระบบที่ดีคือระบบที่ "ไม่เข้า" บ่อยกว่า "เข้า"

## 5. คู่มือความเสี่ยงสำหรับการนำไปใช้จริง (อ่านให้จบก่อนเอาเงินจริงเข้า)

### 5.1 ข้อจำกัดของระบบนี้ (สำคัญมาก)
- นี่คือ **สัญญาณช่วยประกอบการตัดสินใจ** ไม่ใช่ระบบทำนายราคาที่แม่นยำ
- Sentiment จากข่าวพาดหัวไม่สามารถจับ "ปฏิกิริยาจริงของตลาด" ได้ 100% เพราะราคาบางทีเคลื่อนไหว
  ล่วงหน้าไปแล้วก่อนข่าวออก (priced-in) หรือตอบสนองตรงข้ามกับที่คาด
- ราคาจาก Yahoo Finance อาจต่างจากราคาโบรกเกอร์ของคุณเล็กน้อย — ใช้เพื่อเรียนรู้/ทดสอบเท่านั้น
  เมื่อเชื่อมต่อ MT5 แล้วควรใช้ราคาจากโบรกเกอร์โดยตรง
- **ยังไม่ผ่านการ backtest** — สัญญาณข่าว+เทคนิคนี้ยังไม่ได้พิสูจน์ว่าทำกำไรในอดีต ห้ามใช้กับเงินจริง

### 5.2 กฎการบริหารความเสี่ยงที่ต้องทำเสมอ
1. **ไม่เสี่ยงเกิน 1-2% ของทุนต่อ 1 ออเดอร์** (ตั้งไว้ใน `config.py` แล้ว ปรับได้)
2. **ตั้ง Stop-loss ทุกครั้ง** ไม่มีข้อยกเว้น
3. **จำกัดขาดทุนต่อวัน/สัปดาห์** แล้วหยุดเทรดทันทีเมื่อถึงลิมิต (ระบบมีฟังก์ชัน `check_daily_loss_limit` ช่วยเช็คให้)
4. **อย่า revenge trade** — ขาดทุนแล้วรีบเข้าออเดอร์ใหม่เพื่อเอาคืน คือสาเหตุอันดับต้นๆ ที่ทำให้บัญชีล้าง
5. **เริ่มด้วย Demo Account ก่อนเสมอ** อย่างน้อย 1-2 เดือน ก่อนใช้เงินจริง

### 5.3 ข้อควรระวังเรื่อง Leverage
Forex มักมี leverage สูง (50:1 หรือมากกว่า) การใช้ leverage เต็มที่ = ความเสี่ยงเพิ่มตามสัดส่วน
แนะนำใช้ leverage ต่ำกว่าที่โบรกเกอร์เสนอสูงสุดมาก โดยเฉพาะช่วงเริ่มต้น

### 5.4 กฎหมาย/ภาษี
การเทรด Forex กับโบรกเกอร์ต่างชาติมีรายละเอียดทางกฎหมายที่ควรตรวจสอบกับ ก.ล.ต. หรือ ธปท. โดยตรง
เนื้อหานี้ไม่ใช่คำแนะนำทางกฎหมาย

### 5.5 คำเตือนทั่วไป
เอกสารและระบบนี้จัดทำเพื่อการศึกษา/ประกอบการตัดสินใจเท่านั้น ไม่ใช่คำแนะนำการลงทุนจากผู้เชี่ยวชาญที่มีใบอนุญาต
การเทรด Forex มีความเสี่ยงสูง อาจสูญเสียเงินทุนทั้งหมดได้ ควรลงทุนด้วยเงินที่ยอมรับความเสี่ยงจากการสูญเสียได้เท่านั้น
